# 自助火锅竹签计价系统 - 管理员前端设计文档

**项目名称**: 自助火锅竹签自动计价系统  
**目标用户**: 店面管理员  
**设计日期**: 2026-06-30  
**设计状态**: 已确认，待实施

---

## 一、项目概述

### 1.1 业务背景

串串火锅店使用竹签计价模式：顾客用餐后，服务员拍摄签子图片，系统通过 YOLO 自动识别签子数量并计算消费金额。管理员需要通过后台管理员工、调整价格、查看报表、监控实时运营。

### 1.2 核心功能

管理员前端系统需要支持以下功能模块：

1. **实时运营监控** - 查看桌况分布、订单状态、实时数据
2. **员工管理** - 增删改查员工账号、权限管理
3. **价格管理** - 调整签子价格、上架下架菜品
4. **数据报表** - 查看销量排行、营业额统计、桌台绩效

### 1.3 设计目标

- 高效完成管理任务，减少操作步骤
- 克制、清晰、真实产品感的视觉风格
- 暖色系配色，融入火锅行业特色
- 仅支持 PC 端，不做响应式设计

## 四、视觉风格设计

### 4.1 设计原则

**克制、清晰、功能性优先**

- 不追求视觉炫技，每个元素都有明确用途
- 避免模板感：不使用常见后台模板的"蓝橙配色"
- 真实产品感：参考 Stripe Dashboard、Linear、Notion、GitHub

### 4.2 配色方案（暖色系）

#### 浅色模式

```
主题色系（火锅温暖感）
  - 主橙色：      #F97316（炭火橙，主题色）
  - 深橙色：      #EA580C（hover/强调）
  - 浅橙色：      #FED7AA（淡橙背景）
  - 超浅橙：      #FFF7ED（hover 背景）

成功色（火锅红）
  - 成功红：      #DC2626（代替绿色，营业额数字专用）
  - 浅红背景：    #FEE2E2

中性色（保持克制）
  - 主文字：      #1F2937（深灰黑）
  - 次文字：      #6B7280（灰）
  - 辅助文字：    #9CA3AF（浅灰）
  - 边框：        #E5E7EB（浅灰边框）
  - 背景：        #F9FAFB（淡灰背景）
  - 卡片背景：    #FFFFFF（纯白）

状态色（特殊用途）
  - 警告橙：      #F59E0B（清洁中状态）
  - 信息蓝：      #3B82F6（数据图表，克制使用）
```

#### 暗色模式

```
中性色反转
  - 主文字：      #F9FAFB（近白）
  - 次文字：      #D1D5DB（灰白）
  - 辅助文字：    #9CA3AF（灰）
  - 边框：        #374151（深灰边框）
  - 背景：        #111827（深黑蓝）
  - 卡片背景：    #1F2937（深灰）

主题色调整
  - 主橙色：      #F97316（保持不变）
  - 其他状态色保持不变
```

### 4.3 字体系统

#### 字体选择

使用系统字体栈，避免模板感：

```css
font-family: 
  -apple-system, BlinkMacSystemFont,
  "Segoe UI", "Helvetica Neue",
  Arial, sans-serif;

数据/金额专用：
font-family: "SF Mono", "Consolas", monospace;
```

#### 字号节奏

| 层级 | 字号 | 用途 |
|------|------|------|
| H1 页面标题 | 24px | 面包屑下方 |
| H2 区块标题 | 18px | 内容区块 |
| H3 小标题 | 16px | 表格标题 |
| 正文 | 14px | 表格内容、卡片数字 |
| 辅助文字 | 12px | 标签、提示、说明 |
| 按钮 | 14px | 所有按钮统一 |

#### 字重克制

- 常规文字：400（normal）
- 页面标题：600（semibold）
- 重要数据：500（medium）
- **避免使用：700（bold）**

### 4.4 间距节奏

统一间距系统（8px base）：

| 间距级别 | 值 | 用途 |
|----------|-----|------|
| XS | 8px | 小元素间隙 |
| S | 12px | 卡片内间距 |
| M | 16px | 卡片与卡片之间 |
| L | 24px | 区块之间 |
| XL | 32px | 页面顶部到内容 |

## 五、关键组件和状态

### 5.2 桌况卡片组件

#### 设计规格

```

无边框：    纯色块
```

## 六、PC端固定规格

### 6.1 视窗要求

```
最小支持宽度：  1280px
推荐宽度：      1440px - 1920px
最小支持高度：  720px
推荐高度：      900px - 1080px

不支持：        低于 1280px 的设备
提示：          视窗小于 1280px 时显示："请使用桌面浏览器访问"
```

### 6.2 固定布局尺寸

#### 侧边栏

```
宽度：      200px（固定）
不可收起：  始终展开
位置：      固定左侧，不响应视窗变化
```

#### 内容区

```
宽度计算：  视窗宽度 - 侧边栏 200px - 内边距 48px
最小宽度：  1032px
内边距：    左右各 24px
```

#### 顶部栏

```
高度：      64px（固定）
内容：      Logo 左侧固定，用户信息右侧固定
```

### 6.3 组件固定尺寸

#### 表格

```
最小表格宽度：  800px
列宽分配：
  - ID列：       60px
  - 名称列：     120px
  - 类别列：     80px
  - 价格列：     100px
  - 状态列：     80px
  - 操作列：     120px

不响应视窗：    视窗变小时表格宽度不变
                视窗出现横向滚动条
```

#### 卡片

```
统计卡片：      240px × 120px（固定）
桌况卡片：      64px × 64px（固定）
网格排列：      固定 4 列或 6 列，不响应视窗
```

### 6.4 非响应式设计优势

#### 代码层面

优点：
- 无需 media query，CSS 更简洁
- 无需响应式组件切换逻辑
- 布局计算更稳定，减少 bug
- 性能更好，无动态样式重计算

缺点：
- 无法在小屏设备使用
- 未来扩展需单独开发移动端版本

#### 用户体验层面

优点：
- 所有页面布局一致，用户学习成本低
- 数据表格始终清晰可见，无需横向滚动
- 操作按钮位置固定，肌肉记忆建立快

缺点：
- 管理员必须在店内电脑操作
- 无法远程用平板查看数据

### 6.5 CSS 编写规范

#### 禁止使用

```css
/* 禁止：响应式断点 */
@media (max-width: 768px) { }

/* 禁止：响应式宽度 */
max-width: 100vw;
min-width: 320px;

/* 禁止：视窗单位 */
width: 50vw;

/* 禁止：响应式换行 */
flex-wrap: wrap;
```

#### 强制使用

```css
/* 强制：固定 px 单位 */
width: 200px;

/* 强制：固定百分比宽度（基于父容器） */
width: 50%;  /* 父容器固定时 */

/* 强制：固定的 grid 列数 */
grid-template-columns: repeat(4, 240px);
```

### 6.6 视窗检测机制

#### JavaScript 检测

## 八、设计决策总结

### 8.1 核心设计决策

| 决策项 | 选择 | 原因 |
|--------|------|------|
| 视觉风格 | 暖色系克制风格 | 融入火锅行业特色，避免模板感 |
| 配色方案 | 橙色主题+火锅红成功色 | 温暖感，符合火锅氛围 |
| 响应式 | 仅PC端，不做响应式 | 管理后台高频操作，手机体验差 |
| 导航方式 | 侧边栏固定200px | 管理后台需要多层级导航 |
| 数据展示 | 图表+表格结合 | 直观可视化+详细数据 |
| 主题切换 | 暗色/浅色跟随系统 | 用户偏好，提升体验 |
| 登录设计 | 独立登录页 | 安全隔离，专业感 |
| 数据刷新 | 手动刷新按钮 | 管理员主动控制，避免自动干扰 |

### 8.2 不包含的功能

以下功能不在本次设计范围内：

- 服务员端界面（仅管理员端）
- 顾客端界面
- 移动端适配
- 实时推送通知
- 多语言支持
- 高级数据导出（如PDF报告）
