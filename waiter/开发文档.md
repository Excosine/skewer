# 服务员端 — 前端开发文档

> 本文档基于后端 `backend/` 源码（FastAPI + SQLAlchemy + YOLO）编写，所有接口、字段、错误码均与后端实现严格一致。

---

## 1. 概述

### 1.1 项目定位

竹签烤肉店竹签识别计价系统——服务员端 Web 单页应用，用于开单、加菜、拍照识别签子数量、修改数量及结账。

### 1.2 目标用户

服务员（`role = "waiter"`，`role_id = 2`）

### 1.3 技术栈（建议）

| 层 | 方案 |
|---|---|
| 框架 | React 18+ / Vue 3+ |
| 路由 | react-router-dom v6+ / vue-router 4+ |
| HTTP | axios |
| 构建 | Vite |
| 组件库 | Ant Design / Element Plus / 纯 CSS |
| 拍照 | `navigator.mediaDevices.getUserMedia()` + Canvas |

### 1.4 后端地址

```
开发: http://localhost:8000
生产: 由 nginx 反向代理，前端请求 /api/* 即可
```

## 3. 全局架构

### 3.1 axios 实例配置

```ts
// api.ts
import axios from 'axios';

const api = axios.create({ baseURL: '/api' });

// 请求拦截 — 自动带 Token
api.interceptors.request.use(config => {
  const token = localStorage.getItem('token');
  if (token) {
    config.headers.Authorization = `Bearer ${token}`;
  }
  return config;
});

// 响应拦截 — 401 跳登录
api.interceptors.response.use(
  res => res,
  err => {
    if (err.response?.status === 401) {
      localStorage.clear();
      window.location.href = '/login';
    }
    return Promise.reject(err);
  }
);
```

### 3.2 全局状态

| 数据 | 存储位置 | 说明 |
|---|---|---|
| `token` | `localStorage` | JWT，有效期 24h |
| `user` | `localStorage` | `{ username, real_name, role }` |
| 订单数据 | 组件 `state` | 路由内局部状态 |

### 3.3 组件树

```
App
├── LoginPage
└── MainLayout (登录守卫)
    ├── Header     ← 显示 real_name + 退出按钮
    └── Content
        ├── TablePage       (/)
        ├── OrderCreatePage (/orders/new?tableId=1)
        └── OrderDetailPage (/orders/:orderId)
            ├── OrderInfoBar
            ├── ItemList
            │   └── ItemRow × N  (改数 / 删除 / 拍照)
            ├── AddItemModal
            └── CloseOrderFooter
```

## 4. 模块详细设计

---

### 4.1 登录模块

**页面**：`/login`

**API**：`POST /api/auth/login`

**请求**

```ts
// Body (JSON)
interface LoginRequest {
  username: string;
  password: string;
}
```

**响应**

```ts
// 200 OK
interface LoginResponse {
  token: string;       // JWT
  username: string;    // e.g. "waiter01"
  role: "waiter";      // 服务端直接返回 "admin" | "waiter"
  real_name: string;   // e.g. "小李"
}

// 401 Unauthorized
// { detail: "wrong credentials" }
```

**登录流程**

1. 提交用户名密码
2. 成功后保存 `token`、`username`、`role`、`real_name` 到 `localStorage`
3. 跳转 `/`
4. 失败时展示 `"wrong credentials"` 错误提示

**附加 API**：`GET /api/auth/me`

```ts
// 200 OK
interface MeResponse {
  id: number;
  username: string;
  real_name: string;
  role: "admin" | "waiter";
}
```
> 可用于页面刷新时验证 token 有效性。

### 4.2 桌况一览模块

**页面**：`/`

**API**：`GET /api/tables`

**响应**

```ts
// 200 OK — 数组，每项一张桌子
interface TableStatus {
  zone_name:        string;   // "大厅" | "包间" | "露台"
  zone_surcharge:   number;   // 区域加价 (0 / 10 / 5)
  table_code:       string;   // 桌号 "A01"~"C02"
  capacity:         number;   // 座位数 4/6/8
  table_status:     number;   // 0=空闲  1=占用  2=清洁中
  order_id:         number | null;  // 空闲: null, 占用: 订单 ID
  order_no:         string | null;  // 订单编号
  total_count:      number | null;  // 当前签子数
  total_price:      number | null;  // 金额 (结账前为 0)
  waiter_name:      string | null;  // 服务员姓名
  order_created_at: string | null;  // 开单时间 ISO 8601
}
```

**table_status 枚举**

```ts
const TABLE_STATUS = {
  0: { label: '空闲',   color: 'green',  action: 'open' },   // 可开单
  1: { label: '占用',   color: 'red',    action: 'view' },   // 可查看/结账
  2: { label: '清洁中', color: 'yellow', action: 'none' },   // 不可操作
} as const;
```

**展示逻辑**

- 按 `zone_name` 分组渲染（大厅 / 包间 / 露台），每组标题行显示区域名和加价
- 每桌以卡片展示：桌号 + 座位数 + 状态色块
- **空闲桌**：点击 → 跳转 `/orders/new?tableId={table_id}`
- **占用桌**：点击 → 跳转 `/orders/{order_id}`
- **清洁中**：卡片灰显，不可点击
- 占用桌额外显示：订单号、签子数、服务员名、开单时间

### 4.3 开单模块

**页面**：`/orders/new?tableId=1`

**API**：`POST /api/orders`

**请求**

```ts
interface CreateOrderRequest {
  table_id: number;
}
```

**响应**

```ts
// 200 OK
interface CreateOrderResponse {
  id:               number;   // 订单 ID
  order_no:         string;   // 自动生成 "YYYYMMDDHHMMSS-{tableId}"
  table_id:         number;
  zone_surcharge:   number;   // 下单时区域加价快照
  status:           number;   // 0=进行中
  created_at:       string;   // ISO 8601
}

// 400 Bad Request
// { detail: "table not found" }
```

**后端自动行为**

- `table.status` 从 **0(空闲)** → **1(占用)**
- `zone_surcharge` 取自 `table_zones.surcharge`（快照，后续改区域价不影响此单）
- `order_no` 由后端生成（格式：`YYYYmmddHHMMSS-{table_id}`）
- `waiter_id` 从 JWT 中提取，前端无需传入

**页面逻辑**

1. 从 query 参数 `tableId` 读取桌号
2. 展示桌子信息确认弹窗
3. 用户确认后调 API
4. 成功后跳转 `/orders/{id}`

### 4.4 订单详情模块

**页面**：`/orders/:orderId`

这是服务员端核心页面，包含 4 个子区域。

---

#### 4.4.1 订单信息区

**API**：`GET /api/orders/{orderId}`

**响应**

```ts
// 200 OK
interface OrderDetail {
  id:               number;
  order_no:         string;          // 订单编号
  table_id:         number;
  zone_surcharge:   number;          // 区域加价（快照）
  total_count:      number;          // 签子总数（结账前为 0）
  total_price:      number;          // 应付金额（结账前为 0）
  status:           number;          // 0=进行中  1=已结账
  created_at:       string;          // ISO 8601
  items:            OrderItem[];     // 菜品列表
}

// 404 Not Found
// { detail: "order not found" }
```

**展示字段**

| 字段 | 格式示例 |
|---|---|
| 订单号 | `20260630140500-1` |
| 桌号 + 区域 | `A01 (大厅)` |
| 区域加价 | `¥0.00` / `¥10.00` |
| 签子总数 | `30 根` |
| 应付金额 | `¥90.00` |
| 状态 | `进行中` / `已结账` |

---

#### 4.4.2 菜品列表区

```ts
interface OrderItem {
  item_id:      number;    // 明细 ID（注意：不是 "id"）
  skewer_name:  string;    // 菜品名
  count:        number;    // 当前数量
  unit_price:   number;    // 下单时单价快照
  subtotal:     number;    // count × unit_price
}
```

**每行操作按钮**

| 操作 | 说明 | 触发 |
|---|---|---|
| **改数** | `+` / `-` 按钮或直接输入 | 调用 `PUT ...items/{item_id}` |
| **删除** | 移除该菜品行 | 调用 `DELETE ...items/{item_id}` |
| **拍照** | 调起摄像头识别签子数量 | 调用 `POST ...items/{item_id}/scan` |

**注意**：`total_count` 和 `total_price` 只在结账时后端更新，进行中订单这俩字段为 `0`。前端应自己汇总展示：

```ts
// 前端实时计算
const calcCount = (items) => items.reduce((s, i) => s + i.count, 0);
const calcTotal = (items, surcharge) =>
  Math.round((items.reduce((s, i) => s + i.subtotal, 0) + surcharge) * 100) / 100;
```

#### 4.4.3 加菜操作

**API**：`GET /api/menu/categories` + `GET /api/menu/skewers`（进页面时缓存）

**菜单类别**（5 个类别，后端固定种子数据）

```
肉类:   牛肉串 ¥3.00 / 羊肉串 ¥3.50 / 五花肉串 ¥2.50 / 鸡翅 ¥4.00
        鸡胗 ¥2.00 / 掌中宝 ¥3.50 / 烤肠 ¥2.00
蔬菜:   韭菜 ¥1.00 / 金针菇 ¥1.50 / 土豆片 ¥1.00 / 茄子 ¥1.50
        玉米 ¥2.00 / 青椒 ¥1.00
海鲜:   鱿鱼串 ¥4.00 / 虾 ¥5.00 / 生蚝 ¥6.00
豆制品: 豆腐皮 ¥1.50 / 面筋 ¥1.50
主食:   馒头片 ¥1.00 / 年糕 ¥2.00
```

**交互**：弹窗或下拉，按类别分组，点击菜品调用：

```ts
// POST /api/orders/{orderId}/items
interface AddItemRequest { skewer_type_id: number; }

// 200 OK
interface AddItemResponse { id: number; count: number; }  // count 初始为 0

// 如果该菜品已存在 → 返回已有 item_id，不新建
// 400: { detail: "skewer_type not found" }
```

> 加菜后 `count = 0`，服务员需点"拍照识别"或手动改数来填入实际数量。

#### 4.4.4 拍照识别模块

**API**：`POST /api/orders/{orderId}/items/{itemId}/scan`

**请求**：`multipart/form-data`，字段名 `image`，文件类型 PNG/JPEG。

**响应**

```ts
// 200 OK
interface ScanResponse {
  detected_count:  number;    // YOLO 识别到的签子数
  confidence_avg:  number;    // 平均置信度 (0~1)
}
```

**后端行为**

1. 接收图片 → 保存为临时文件
2. 调用 `StickDetector.detect()` (YOLO 模型 `weights/model.pt`)
3. 统计检测到的签子数 → 自动 `UPDATE order_items SET count = X`
4. 删除临时文件
5. 返回识别结果

**前端交互**

| 步骤 | 说明 |
|---|---|
| 1. 点击拍照 | 打开摄像头 / 调用相册 |
| 2. 拍照确认 | Canvas 截帧，转为 File/Blob |
| 3. 上传 | `FormData.append('image', file)` |
| 4. Loading | 等待中展示加载动画 |
| 5. 结果展示 | 显示 `detected_count` + `confidence_avg`，更新列表中的数量 |

```ts
// 上传伪代码
const formData = new FormData();
formData.append('image', blob, 'photo.jpg');

const res = await api.post<ScanResponse>(
  `/api/orders/${orderId}/items/${itemId}/scan`,
  formData,
  { headers: { 'Content-Type': 'multipart/form-data' } }
);
// 用 res.data.detected_count 更新 item.count
```

#### 4.4.7 结账

**API**：`POST /api/orders/{orderId}/close`

```
200 OK  →  { status: "ok" }
400    →  { detail: "order not found" }
```

**后端自动行为**

| 操作 | 说明 |
|---|---|
| `order.status` | 0 → **1（已结账）** |
| `order.paid_at` | 设为当前时间 |
| `order.total_count` | `SUM(order_items.count)` |
| `order.total_price` | `SUM(order_items.subtotal) + zone_surcharge` |
| `table.status` | 1 → **2（清洁中）** |

**交互**：点击结账 → 弹出确认框（展示汇总） → 确认 → 成功后跳回 `/`。

**确认框汇总示例**

```
桌号: A01
签子总数: 30 根
菜品金额: ¥85.00
区域加价: ¥0.00 (大厅)
—————————————
合计: ¥85.00
```

## 6. 数据字典（完整）

### 6.1 桌子状态

| 值 | 状态 | 颜色 | 前端操作 |
|---|---|---|---|
| `0` | 空闲 | 🟢 绿 | 可开单 |
| `1` | 占用 | 🔴 红 | 可查看/结账 |

### 6.5 错误码速查

| 状态码 | 场景 |
|---|---|

| `404` | 订单不存在 / 明细不存在 |
